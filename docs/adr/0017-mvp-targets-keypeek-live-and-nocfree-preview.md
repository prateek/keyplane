# MVP targets KeyPeek live state and NocFree preview

The MVP will prove the live overlay with one KeyPeek-supported QMK/Vial keyboard using the KeyPeek firmware module as the authoritative runtime path. The user's NocFree/Vial workflow will be the first import-and-preview target, giving the project an immediately useful non-firmware-module path while keeping stock Vial behavior classified as Best-Effort Preview rather than live layer sync.
